## 3. โมดูลฝ่ายขาย

### 3.1 การจัดการลูกค้า (Sales → การจัดการลูกค้า)
**ใหม่**: รายชื่อลูกค้าเชิงฝ่ายขาย แสดงจำนวนผู้ติดต่อ / ดีล / pipeline / ติดต่อล่าสุด
- หน้ารายละเอียด → **OCR นามบัตร** (Tesseract.js)
  1. อัปโหลดรูปนามบัตร
  2. กด "OCR Read"
  3. ตรวจสอบข้อมูลที่สกัด (ชื่อ/บริษัท/โทร/เมล) แล้วบันทึก
- รูปจะถูกบีบอัดอัตโนมัติ (สูงสุด 1200px, JPEG 85%) + thumbnail (สูงสุด 400px)

### 3.2 การจัดการดีล (Sales → ดีล)
**ใหม่**: ตัวกรองขยาย
- ลูกค้า (dropdown)
- สถานะ (21 ขั้น: Lead Identified ~ Closed Won)
- ผู้ขาย
- ชื่อดีล (บางส่วน)
- จำนวนเงิน min~max
- โอกาส (0~100%)
- ค้นหาอิสระ (Deal No / PJ No)

หน้าจอแก้ไข:
- **อัตรากำไรประเมิน (%)** คำนวณอัตโนมัติจากหมวดโซลูชัน
  - IT = 100% / OT, FA = 10% / Hardware, License, Other = 0%
- **โครงการที่เกี่ยวข้อง** เลือกจากรายการคำสั่งขาย
- **ชื่อผู้ขาย** ยกเลิก (รวมกับ dropdown ผู้ขาย)

### 3.3 Activity Log
**ใหม่**: ฟิลด์บังคับเพิ่มเติม
- **ผู้ติดต่อ** (บังคับ, เลือกจากมาสเตอร์ผู้ติดต่อลูกค้า)
- **เวลาเริ่ม / เวลาสิ้นสุด** → ระยะเวลาคำนวณอัตโนมัติ
- **บันทึกประชุม** (เดิม: คำอธิบาย)
- **การกระทำถัดไป / วันที่** (บังคับ)

### 3.4 ใบเสนอราคา
- สร้างใหม่ → **วันหมดอายุ = วันออก + 1 เดือน** อัตโนมัติ
- Multi-step approval (Draft → Internal Review → Manager → Director)
- PDF แสดงชื่อและตราประทับผู้อนุมัติ
- เงื่อนไขการชำระเงินดึงจาก `payment_term_installments` พร้อม % และ EN description

### 3.5 ลงทะเบียนคำสั่งขาย
**ใหม่**:
- แปลงจากดีล → ป้องกันลงซ้ำอัตโนมัติ
- สถานะรวมเป็น 3 ค่าเท่านั้น:
  - `CONFIRMED` (ลงทะเบียน)
  - `INVOICED` (อัตโนมัติเมื่อออกใบแจ้งหนี้ AR)
  - `CANCELLED`
- **ปุ่มยกเลิกเฉพาะผู้ดูแลระบบ**

## 4. โมดูลจัดซื้อ

### 4.1 รายการใบสั่งซื้อ
**ใหม่**: ตัวกรองขยาย — สถานะ, ผู้จัดจำหน่าย dropdown, ช่วงวันที่, ค้นหาอิสระ

### 4.2 สร้างใบสั่งซื้อ
- เลือกผู้จัดจำหน่าย → สินค้า → จำนวน/ราคา → VAT อัตโนมัติ
- PDF ใบสั่งซื้อที่อนุมัติแล้วแสดงตราประทับ

---

## 5. โมดูลบัญชี / AR / AP

### 5.1 ใบแจ้งหนี้ AR **ใหม่**
- ส่วนหัว: **เลือกจากคำสั่งขาย** dropdown
  - โหลด → ลูกค้า / รายการ / **เงื่อนไขชำระ** / **วันครบกำหนด** เติมอัตโนมัติ
  - วันครบกำหนด = วันที่ใบแจ้งหนี้ + Credit Days
- **เงื่อนไขชำระล็อกจากใบเสนอราคา** (เฉพาะผู้ดูแลระบบแก้ไขได้)
- บันทึก → สถานะคำสั่งขายเปลี่ยนเป็น `INVOICED` อัตโนมัติ
- หน้ารายละเอียด: สรุปยอด / รายการ / ประวัติการชำระ / ดาวน์โหลด PDF

### 5.2 ใบแจ้งหนี้ AP
- สร้างจากใบสั่งซื้อหรือกรอกด้วยตนเอง

### 5.3 การรับชำระเงิน
- บันทึกการชำระแบบบางส่วน / จัดสรรให้ใบแจ้งหนี้หลายใบ

## 6. การจัดการมาสเตอร์

| มาสเตอร์ | เนื้อหา |
|----------|---------|
| ลูกค้า | CUS-XXXX (ห้ามซ้ำ) / 3 ภาษา / ผู้ติดต่อ + นามบัตร |
| ผู้จัดจำหน่าย | SUP-XXXX / อัตรา WHT |
| สินค้า | ITM-XXXX / เวอร์ชันตามช่วงเวลา |
| ผังบัญชี | ตาม TFRS |
| เงื่อนไขชำระ | รองรับงวด (เช่น 50% DP + 40% Install + 10% HO, Net30 ละงวด) |
| ธนาคาร | CRUD |
| ฝ่าย / แผนก | 3 ภาษา (13 แผนก) |
| หมวดโซลูชัน | **ใหม่**: 32 รายการ + กลุ่ม (IT/OT/FA/HW/License/Other) + อัตรากำไรประเมิน |
| สถานะดีล | 21 รายการ |
| **อัตราแลกเปลี่ยน** | **ใหม่**: 16 สกุลเงินข้ามกัน, ช่วงวันที่, สร้างอัตรากลับอัตโนมัติ |
| **สิทธิ์การใช้งาน** | **ใหม่ (admin)**: เมทริกซ์ checkbox |
| **บันทึกเปลี่ยนแปลง** | **ใหม่ (admin)**: ประวัติการเปลี่ยนแปลงทั้งระบบ |

## 7. สิทธิ์และบันทึกเปลี่ยนแปลง

### 7.1 สิทธิ์การใช้งาน (admin)
- มาสเตอร์ → สิทธิ์การใช้งาน
- Role × Permission checkbox matrix
- ADMIN role เปิดทั้งหมดโดยบังคับ

### 7.2 บันทึกการเปลี่ยนแปลง (admin)
- มาสเตอร์ → บันทึกการเปลี่ยนแปลง
- Filter: ตาราง / การกระทำ / ผู้ใช้ / ช่วงวันที่ / ค้นหา
- หน้ารายละเอียดแสดง **ค่าเดิม vs ค่าใหม่** (แถวเปลี่ยนแปลง = พื้นหลังเหลือง)

## 10. การแก้ปัญหา

| อาการ | วิธีแก้ |
|-------|---------|
| เข้าสู่ระบบไม่ได้ | ตรวจตาราง `users`, รีเซ็ตรหัสผ่านด้วย `database/reset_passwords.php` |
| PDF ว่างเปล่า | ตรวจ console, ตรวจโลโก้, เปิด GD/fileinfo extension |
| OCR ไม่ทำงาน | ตรวจอินเทอร์เน็ต (CDN), ตรวจ Tesseract error ใน console |
| ภาษาไทยเพี้ยน | encoding DB = UTF8, เปิด mbstring |
| 500 error | ตรวจ PHP error log |
| SQLSTATE[25P02] | transaction ก่อนหน้าล้มเหลว; รีสตาร์ทเซิร์ฟเวอร์ |
| รหัสลูกค้าซ้ำ | ถูกบล็อกโดย `uniq_customer_code_active` |

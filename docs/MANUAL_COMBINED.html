第 1 部: インストール手順

1.1 前提条件

項目推奨バージョン
OSWindows 10/11 または Ubuntu 20.04+
PHP8.2 以上
PostgreSQL15 以上
ブラウザChrome / Edge 最新版
メモリ4 GB 以上
ディスク10 GB 以上の空き

1.2 PHP のセットアップ

  1. PHP 8.2+ をインストール (Windows は scoop install php 推奨)
  2. php.ini で以下を有効化: 
    extension=pdo_pgsql
extension=pgsql
extension=mbstring
extension=gd
extension=fileinfo
date.timezone = "Asia/Bangkok"
  3. php -v でバージョン確認、php -m | grep pgsql で拡張有効化確認

1.3 PostgreSQL のセットアップ

  1. PostgreSQL 15+ をインストール
  2. データベース作成: psql -U postgres -c "CREATE DATABASE pegasus_erp ENCODING 'UTF8';"
  3. 文字コードは UTF8 必須 (タイ語・日本語サポート)

1.4 ソースコードの配置

ソース zip を任意のフォルダに展開 (例: C:\Pegasus_ERP_R1\)

Pegasus_ERP_R1\
├── config\          — DB設定
├── controllers\     — コントローラ
├── core\            — コアフレームワーク
├── database\        — スキーマ・SQL
├── lang\            — 言語ファイル (en/ja/th)
├── models\          — モデル
├── public\          — Webルート
├── views\           — テンプレート
├── uploads\         — アップロード領域
└── backups\         — DBバックアップ

1.5 DB 接続設定

config/database.php を編集するか環境変数を設定:

環境変数既定値
DB_HOSTlocalhost
DB_PORT5432
DB_NAMEpegasus_erp
DB_USERpostgres
DB_PASSpostgres

1.6 DB 初期化

既存バックアップからリストア:

psql -U postgres -d pegasus_erp -f backups\pegasus_erp_YYYYMMDD_HHMMSS.sql

新規構築: setup-database.bat を実行

1.7 サーバー起動

start-server.bat
または
php -S localhost:8080 -t public/

ブラウザで http://localhost:8080 にアクセス

1.8 初回ログイン

項目値
URLhttp://localhost:8080/login
ユーザー名admin
初期パスワードadmin123
重要: ログイン後、必ずパスワードを変更してください。

2.3 基本フロー: 見積書 → 受注 → 発注 → 入荷

  1. 案件登録 (営業 → 案件): 顧客・案件名・見込度・金額を入力
  2. 見積書作成: 案件と紐付け、明細・支払条件を入力
  3. 承認依頼 → 承認: Manager/Director が承認。承認者名+スタンプが PDF に表示
  4. 受注化: 承認済み見積書から Sales Order を生成
  5. 発注: 仕入先への発注書を作成
  6. 入荷処理: 入荷を記録して在庫更新
  7. AR 請求書: 受注から請求書生成 → 入金処理

2.4 案件検索

案件一覧画面のフィルタ:

  • 顧客: プルダウン選択
  • ステータス: Lead Identified ~ Closed Won 等
  • 営業担当: プルダウン
  • 案件名: 部分一致
  • 金額: min ~ max
  • 見込度: 0 ~ 100 %
  • フリー検索: Deal No / PJ No

2.5 PDF プレビュー / 印刷

  • 「PDF」ボタンで新規タブプレビューが開く
  • 上部「Print / Save PDF」で印刷ダイアログ表示
  • 「Close」でタブを閉じる
  • ファイル名 = 文書番号 (例: QT-2026-0001.pdf)

2.6 承認ワークフロー

  1. 作成者が「承認依頼」→ ステータス PENDING_APPROVAL
  2. Manager / Director が「承認」ボタン押下
  3. 承認後、PDF に承認者氏名 + スタンプ自動表示

2.7 言語切替

右上ユーザーメニュー → 日本語 / English / ไทย

2.8 バックアップ

# 定期バックアップ
pg_dump -U postgres -d pegasus_erp -f backups\pegasus_erp_YYYYMMDD.sql
pg_dump -U postgres -d pegasus_erp -Fc -f backups\pegasus_erp_YYYYMMDD.dump

# リストア
psql -U postgres -d pegasus_erp -f backups\pegasus_erp_YYYYMMDD.sql
pg_restore -U postgres -d pegasus_erp -c backups\pegasus_erp_YYYYMMDD.dump

2.3 Flow พื้นฐาน

  1. สร้างดีล: กรอกลูกค้า/ชื่อดีล/โอกาส/จำนวนเงิน
  2. สร้างใบเสนอราคา: เชื่อมกับดีล, กรอกรายการ/เงื่อนไขชำระ
  3. ขออนุมัติ → อนุมัติ: Manager/Director อนุมัติ
  4. แปลงเป็นคำสั่งขาย: สร้าง Sales Order จากใบเสนอราคา
  5. ใบสั่งซื้อ: สร้าง PO ไปยังผู้ขาย
  6. รับสินค้า: บันทึก GR → อัปเดตสต๊อก
  7. AR Invoice: ออกใบแจ้งหนี้ → รับชำระ

2.4 การค้นหาดีล

  • ลูกค้า: dropdown
  • สถานะ: dropdown (Lead Identified ~ Closed Won)
  • ผู้ขาย: dropdown
  • ชื่อดีล: partial match
  • จำนวนเงิน: min ~ max
  • โอกาส: 0 ~ 100 %
  • ค้นหาอิสระ: Deal No / PJ No

2.5 PDF Preview / พิมพ์

2.6 Workflow การอนุมัติ

  3. หลังอนุมัติ ชื่อผู้อนุมัติ + ตราประทับแสดงบน PDF

## 5. มาสเตอร์ (Master Data)

### 5-1. 👤 ลูกค้า (Customers)
**เพิ่มลูกค้าใหม่**:
1. มาสเตอร์ → ลูกค้า → **+ เพิ่มลูกค้า**
2. กรอก: ชื่อ / Tax ID / ที่อยู่ / เบอร์โทร / อีเมล / วงเงินเครดิต / เงื่อนไขชำระเงิน
3. รหัสจะถูกสร้างอัตโนมัติ (เช่น `CUS-0465`)
4. **พนักงานทั่วไปสร้าง** → สถานะ `PENDING` (รอการอนุมัติ)
5. **Manager เข้าไปที่ คิวการอนุมัติ → อนุมัติลูกค้า** → อนุมัติ/ปฏิเสธ

### 5-2. 🏭 ผู้จัดจำหน่าย (Suppliers)
ใช้วิธีเดียวกันกับลูกค้า รหัส = `SUP-XXXX`

### 5-3. 📦 สินค้า (Items)
กำหนด: รหัส / ชื่อ / หน่วย / ต้นทุน / ราคาขาย / ภาษี

### 5-4. 💳 เงื่อนไขชำระเงิน (Payment Terms)
สนับสนุนการแบ่งงวด เช่น:
- **NET30/60/90**: ชำระเต็มจำนวนเครดิต 30/60/90 วัน
- **DEP50**: มัดจำ 50% / ที่เหลือ 50% หลังส่งมอบ
- **DP50-MILE4**: 50% DP / 20% ออกแบบเสร็จ / 20% ติดตั้ง / 10% ส่งมอบ

### 5-5. 🏦 ธนาคารบริษัท (Company Bank Accounts)
บัญชีธนาคารของบริษัทที่จะแสดงบนใบแจ้งหนี้ลูกค้า:
- ชื่อธนาคาร (EN/TH) / สาขา / เลขที่บัญชี / ชื่อบัญชี / สกุลเงิน / SWIFT
- ตั้งเป็น **Default ต่อสกุลเงิน** ได้ 1 บัญชี

### 5-6. 💱 อัตราแลกเปลี่ยน (Exchange Rates)
- 16 สกุลเงิน: THB / USD / JPY / EUR / CNY / SGD / MYR / VND / PHP / KRW / TWD / HKD / INR / GBP / AUD / IDR
- ช่วงเวลาผล (Effective From / To)
- รหัส API: `GET /api/exchange-rate?from=USD&to=THB&date=2026-04-22`

### 5-7. 🏷 หมวดหมู่โซลูชัน (Solution Categories)
32 หมวด แบ่งเป็น:
- **IT Solution**: กำไรประเมิน 100%
- **OT / FA**: กำไรประเมิน 10%
- **Hardware / License / Other**: 0%

## 6. โมดูลการขาย (Sales)

### 6-1. 📂 การจัดการดีล
**สร้างดีลใหม่**:
1. การขาย → การจัดการดีล → **+ ดีลใหม่**
2. กรอก: ชื่อดีล / ลูกค้า / ความน่าจะเป็น / หมวดโซลูชัน / ผู้รับผิดชอบ / จำนวนเงินคาด
3. **Evaluation Profit %** คำนวณอัตโนมัติจากหมวดโซลูชัน
4. กด **บันทึก**

**การกรองดีล**:
- ตามลูกค้า / สถานะ / ผู้ขาย / ชื่อดีล / จำนวนเงิน / ความน่าจะเป็น

**Kanban View**: ดีลแบ่งตามสถานะในรูปแบบการ์ด ลากเพื่อเปลี่ยนสถานะ

### 6-2. 📝 ใบเสนอราคา
**สร้างใบเสนอราคา**:
1. ใบเสนอราคา → **+ ใบเสนอราคาใหม่**
2. เลือก: ลูกค้า / วันที่ / ผู้ติดต่อ / **วันที่หมดอายุ = วันที่ออก + 1 เดือน (อัตโนมัติ)**
3. เงื่อนไขชำระเงิน: เลือกจากมาสเตอร์
4. เพิ่มรายการ: หมวด/สินค้า/จำนวน/ราคา
5. **🗓 กำหนดการตรวจรับ**:
   - คลิก "📋 โหลดจากเงื่อนไขชำระ" → ระบบสร้างงวดจากเงื่อนไขชำระเงิน
   - หรือ "+ เพิ่มแถว" ด้วยตนเอง
   - กรอก: Seq / คำอธิบาย / % / วันที่ตรวจรับ / สถานะ
6. **บันทึกเป็น DRAFT** หรือ **ขออนุมัติ**

**การอนุมัติ**:
1. STAFF → **ขออนุมัติ** → สถานะ `PENDING_APPROVAL`
2. Manager → คิวการอนุมัติ → **✓ อนุมัติ** / **✕ ปฏิเสธ**

### 6-3. 🧾 คำสั่งขาย (Sales Orders)
สร้างจากใบเสนอราคาที่ APPROVED:
1. ใบเสนอราคา → รายละเอียด → **แปลงเป็นคำสั่งขาย**
2. สถานะ: `CONFIRMED` → `INVOICED` (อัตโนมัติเมื่อออกใบแจ้งหนี้) → `CANCELLED` (Admin เท่านั้น)

### 6-4. 📋 บันทึกกิจกรรม (Activity Log)
**บันทึกการติดต่อลูกค้า**:
1. การขาย → บันทึกกิจกรรม
2. ฟอร์มด้านบน:
   - **วันที่ / ผู้ขาย / ประเภท** (โทร/เยี่ยม/ประชุม/อีเมล/อื่นๆ)
   - **ลูกค้า / ผู้ติดต่อ** (จำเป็น)
   - เวลาเริ่ม / เวลาจบ → ระบบคำนวณระยะเวลาอัตโนมัติ
   - **เรื่อง / บันทึกประชุม**
   - **การกระทำถัดไป / วันครบกำหนด** (จำเป็น)
3. กด **เพิ่มกิจกรรม**

**🔁 ฟังก์ชัน Follow-up**:
- ในแถวกิจกรรม → กด **🔁 ติดตาม**
- ระบบ **คัดลอกแถว** (ลูกค้า/ผู้ติดต่อ/ประเภท) พร้อมเชื่อมโยง parent-child
- ป้ายแสดงจำนวน follow-up (เช่น 🔁 ติดตาม ③)

### 6-5. 📈 Pipeline
ภาพรวมดีลทั้งหมด แบ่งตามสถานะและจำนวนเงิน

## 10. AR / AP (ลูกหนี้/เจ้าหนี้)

### 10-1. ใบแจ้งหนี้ AR
**สร้างใบแจ้งหนี้**:
1. AR → ใบแจ้งหนี้ → **+ ใหม่**
2. **เลือกจากคำสั่งขาย** (dropdown) → ระบบโหลด: ลูกค้า / รายการ / เงื่อนไขชำระ / วันครบกำหนด
3. **🔀 แยกใบแจ้งหนี้ (Installments)**:
   - ถ้าเงื่อนไขชำระมีหลายงวด (เช่น 50/20/20/10)
   - เลือก **checkbox** งวดที่ต้องการออกใบแจ้งหนี้
   - หรือกด **"เลือกทั้งหมดที่เหลือ"** เพื่อออกยอดคงเหลือ
   - รายการจะคำนวณเป็นสัดส่วน
4. **บันทึก** → PDF สร้างอัตโนมัติ

**PDF รูปแบบใบเสร็จรับเงิน/ใบกำกับภาษี**:
- รูปแบบตาม Thai Tax Invoice
- โลโก้บริษัท / เลขที่บัญชี / ยอดเงินเป็นคำ (ภาษาไทย)
- ช่องลายเซ็น 4 ช่อง (Bill Collector / Goods Deliver / Goods Receiver / Authorized)

### 10-2. รับชำระ (AR Payments)
บันทึกการรับชำระจากลูกค้า → ตัดเข้าใบแจ้งหนี้

### 10-3. AP (เจ้าหนี้)
เหมือน AR แต่สำหรับการจ่ายผู้จัดจำหน่าย

## 13. การผลิต (Production / MRP)

### MO (ใบสั่งผลิต)

### MRP (วางแผนความต้องการวัตถุดิบ)

## 15. กระแสเงินสด (Cashflow)

### 15-1. พยากรณ์กระแสเงินสด
**สูตรการคำนวณ**:
- ใบเสนอราคา × ความน่าจะเป็น × เดือนที่คาดรับเงิน
- แยกตามงวดการชำระ (installment)
- ต่อเดือน (Apr-Mar)

**การอ่านตาราง**:
- **แถว**: สถานะดีล (ความน่าจะเป็น 5% → 100%)
- **คอลัมน์**: เดือน
- **Weighted**: จำนวนเงิน × ความน่าจะเป็น%

### 15-2. กระแสเงินสดจริง
AR รับแล้ว - AP จ่ายแล้ว = เงินสดสุทธิ (รายเดือน)

## 16. การวิเคราะห์และรายงาน

### 16-1. แดชบอร์ดการวิเคราะห์
- **ใบเสนอราคา**: ตามความน่าจะเป็น / ลูกค้า / โซลูชัน / ช่วงเวลา
- **ใบสั่งซื้อ**: ตามผู้จัดจำหน่าย / สถานะ / ช่วงเวลา

### 16-2. รายงาน
หน้ารายการรายงาน เชื่อมไปยังหน้าต่างๆ:
- Sales → Quotation Analytics
- Financial → P&L / BS / Ledger / Cashflow
- HR → Attendance / Leave / Headcount
- Inventory → Stock
- Tax → PND3 / PND53 / PP30 (กำลังพัฒนา)

## 17. KPI การขาย

### 17-1. ดู KPI Dashboard
การขาย → **🎯 แดชบอร์ด KPI**

แสดงรายพนักงาน:
- **Contact**: จำนวนกิจกรรม (เทียบกับเป้า)
- **Meeting**: จำนวนประชุม
- **Profit**: กำไรจริง (Closed Won)
- **% การบรรลุเป้า**: เขียว ≥100% / ส้ม ≥80% / น้ำเงิน ≥50% / แดง <50%

### 17-2. ตั้งค่า KPI Target Master (Manager+ เท่านั้น)
**กำหนดเป้า**:
1. Master → **⚙ KPI Target Master**
2. เลือกพนักงาน / ปีบัญชี
3. กรอก:
   - **เป้ากำไรรายปี** (THB)
   - **กำไรต่อคำสั่งซื้อ** (THB)
   - **อัตราปิด %**
   - **อัตรานัด %**
4. ระบบคำนวณ:
   - จำนวนคำสั่งซื้อ = เป้ากำไร ÷ กำไรต่อคำสั่ง
   - จำนวนประชุม = จำนวนคำสั่ง ÷ อัตราปิด
   - จำนวน Contact = จำนวนประชุม ÷ อัตรานัด
5. **สัดส่วนรายเดือน %**: ปรับได้เอง (เริ่มต้น เม.ย. 4% / อื่นๆ 12%)

## 18. คิวการอนุมัติ (Approval Queue)

### สำหรับ Manager+ เท่านั้น

**4 คิวแยก**:
| คิว | URL | ข้อมูล |
|---|---|---|
| 👤 ลูกค้า | `/approvals/customers` | ลูกค้าใหม่ที่พนักงานสร้าง |
| 🏭 ผู้จัดจำหน่าย | `/approvals/suppliers` | ผู้จัดจำหน่ายใหม่ |
| 📝 ใบเสนอราคา | `/approvals/quotations` | ใบเสนอราคาที่ขออนุมัติ |
| 📦 ใบสั่งซื้อ | `/approvals/purchase-orders` | PO ที่ขออนุมัติ |

**การอนุมัติ**:
1. คลิกคิวที่ต้องการ → รายการสถานะ **PENDING** แสดง
2. กด **👁 ดู** เพื่อตรวจสอบรายละเอียด
3. กด **✓ อนุมัติ** → สถานะเปลี่ยนเป็น `APPROVED` → ใช้งานได้
4. กด **✕ ปฏิเสธ** → เอกสารกลับเป็น `DRAFT` เพื่อแก้ไขใหม่

**กรองตามสถานะ**: PENDING / APPROVED / REJECTED

## 19. การจัดการผู้ใช้ (Admin)

### สำหรับ ADMIN เท่านั้น

**หน้า**: ผู้ดูแลระบบ → **👥 การจัดการผู้ใช้** (`/admin/users`)

### ฟีเจอร์
| ฟังก์ชัน | คำอธิบาย |
|---|---|
| **+ ผู้ใช้ใหม่** | สร้าง: อีเมล / รหัสผ่าน / บทบาท / เชื่อมโยงพนักงาน |
| **⚙ แก้ไข** | เปลี่ยนบทบาท / ปิด-เปิดใช้งาน / เชื่อมโยงพนักงาน |
| **🔑 PW** | รีเซ็ตรหัสผ่าน 1 ผู้ใช้ |
| **🔑 รีเซ็ตทั้งหมด** | รีเซ็ตรหัสผ่านของผู้ใช้ทุกคนเป็นรหัสเดียวกัน |
| **🗑 ลบ** | ปิดการใช้งาน (soft delete) |

### ความปลอดภัย
- ป้องกันลบตนเอง
- ป้องกันปิด ADMIN คนสุดท้าย
- รหัสผ่านต้อง ≥ 6 ตัวอักษร
- ตรวจสอบอีเมล/username ซ้ำ

## 20. ปัญหาที่พบบ่อย

| อาการ | วิธีแก้ |
|---|---|
| ล็อกอินไม่ได้ | ล้าง cache (Ctrl+Shift+R) หรือใช้ Incognito Window |
| PDF เบลอ / ไม่แสดง | ตรวจสอบว่า PHP `extension=gd` เปิดใช้งาน |
| อัปโหลดรูปนามบัตรเจอ error | ตรวจสอบ `extension=fileinfo` เปิดใช้งาน |
| ภาษาไทยเพี้ยน | DB encoding ต้องเป็น `UTF8` |
| ไม่เห็นเมนู "การจัดการผู้ใช้" | เข้าสู่ระบบด้วยสิทธิ์ ADMIN เท่านั้น |
| ไม่เห็นคิวการอนุมัติ | ต้องเป็น MANAGER หรือ ADMIN |
| เปลี่ยนสถานะใบเสนอราคาแล้ว error | รัน `fix_missing_columns.sql` และ ตรวจสอบ constraint |
| Sidebar เลื่อนขึ้นบนหลังคลิก | ล้าง cache แล้วลองใหม่ (ระบบจะบันทึกตำแหน่ง scroll อัตโนมัติ) |

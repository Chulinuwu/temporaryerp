## 3. 営業モジュール

### 3.1 顧客管理 (営業 → 顧客管理)
**NEW**: 営業向け顧客一覧。担当者数/案件数/パイプライン合計/最終接触日を表示。
- 顧客詳細 → **名刺 OCR 取り込み** (Tesseract.js)
  1. 名刺画像アップロード
  2. 「OCR で読み取り」をクリック
  3. 抽出された氏名/会社名/電話/メール等を確認して保存
- 画像は自動圧縮 (最大1200px、JPEG 85%) + サムネイル (最大400px)

### 3.2 案件管理 (営業 → 案件)
**NEW**: フィルタ拡張
- 顧客 (ドロップダウン)
- ステータス (21段階: Lead Identified 〜 Closed Won)
- 営業担当
- 案件名 (部分一致)
- 金額 min〜max
- 見込度 (0〜100%)
- フリー検索 (Deal No / PJ No)

案件編集画面:
- **評価利益率 (%)** はソリューションカテゴリから自動算出
  - IT = 100% / OT, FA = 10% / Hardware, License, Other = 0%
- **関連プロジェクト** は受注一覧から選択
- **営業名** 欄は廃止 (営業担当ドロップダウンに統一)

### 3.3 活動ログ
**NEW**: 必須項目追加
- **コンタクト者** (必須、顧客担当者マスタから選択)
  - マスタに未登録の場合は、顧客詳細画面で先に登録
- **開始時間 / 終了時間** → **所要時間** を自動算出
- **議事** (旧: 説明): 会議内容を記録
- **次のアクション / 次のアクション日** (必須入力)

### 3.4 見積書
- 新規作成 → **有効期限は自動的に発行日 + 1ヶ月** で設定 (発行日変更時も追従)
- 多段承認フロー (Draft → Internal Review → Manager 承認 → Director 承認)
- 承認済み見積書は PDF に承認者氏名 + スタンプ画像が表示
- 支払条件は `payment_term_installments` テーブルから % + EN 説明を展開

### 3.5 受注登録
**NEW**: #11/#12
- 案件 → 受注変換時、**重複登録を自動防止**
- ステータスは以下 3 種に統一 (手動変更不可):
  - `CONFIRMED` (受注登録時)
  - `INVOICED` (AR 請求書作成時、自動遷移)
  - `CANCELLED` (キャンセル時)
- **キャンセルボタンは管理者のみ表示**

## 4. 購買モジュール

### 4.1 発注一覧
**NEW**: 検索機能拡張
- ステータスドロップダウン
- 仕入先ドロップダウン (発注ありの仕入先のみ)
- 日付範囲 (from / to)
- フリー検索 (PO No / 仕入先名)

### 4.2 発注書作成
- 仕入先 → 品目選択 → 数量・単価入力 → VAT 自動計算
- 承認済み発注書は PDF に承認者スタンプ表示
- 支払条件は payment_terms マスタから選択

## 5. 会計・AR/AP モジュール

### 5.1 売掛請求書 (AR Invoice) **NEW**
- 新規作成時、画面上部に **受注選択** ドロップダウン
  - 受注を選ぶと顧客・明細・**支払条件**・**支払期日** が自動入力
  - 支払期日 = 請求日 + Credit Days (支払条件から自動取得)
- **支払条件は見積書から固定** (管理者のみ編集可能、一般ユーザーはグレーアウト)
- 保存時、受注のステータスが自動で `INVOICED` に遷移
- 詳細画面: 金額サマリ + 明細 + 入金履歴 + PDF ダウンロード

### 5.2 買掛請求書 (AP Invoice)
- 発注書から自動展開、または手動入力
- AP 支払管理 (部分支払対応)

### 5.3 入金管理
- 請求書に対する部分入金を記録
- 複数請求書への按分 (ar_payment_allocations)

## 6. マスタ管理

| マスタ | 内容 |
|--------|------|
| 顧客 | CUS-XXXX (重複禁止) / EN・JP・TH 3言語名 / 担当者リスト + 名刺画像 |
| 仕入先 | SUP-XXXX / WHT税率設定 |
| 品目 | ITM-XXXX / 有効期間バージョニング |
| 勘定科目 | TFRS準拠 Chart of Accounts |
| 支払条件 | 分割明細対応 (例: 50%DP + 40%据付 + 10%引渡, 各Net30) |
| 銀行 | CRUD |
| 事業部 / 部署 | 3言語名 (13部門) |
| ソリューションカテゴリ | **NEW**: 32種 + カテゴリグループ (IT/OT/FA/Hardware/License/Other) + 評価利益率 |
| 案件ステータス | 21種 (Phase 1〜5 + Lost/On Hold) |
| **為替マスタ** | **NEW**: 16通貨相互、有効期間設定、逆方向自動生成 |
| **権限設定** | **NEW (管理者)**: ロール×権限のチェックボックスマトリクス |
| **変更ログ** | **NEW (管理者)**: 全マスタ・トランザクションの変更履歴 |

### 6.1 為替マスタ
- メニュー: マスタ → **為替マスタ**
- 通貨ペア + レート + 有効期間(From/To) を登録
- 逆方向レート自動生成オプション (USD→THB と THB→USD)
- API: `GET /api/exchange-rate?from=USD&to=THB&date=2026-04-17`

## 7. 権限と監査ログ

### 7.1 権限設定 (管理者のみ)
- メニュー: マスタ → **権限設定**
- ロール × 権限のチェックボックスマトリクス
- 権限カテゴリ: sales / purchasing / inventory / accounting / ar / ap / hr / payroll / expense / production / master / admin
- ADMIN ロールは全権限強制 ON (無効化不可)

### 7.2 変更ログビューア (管理者のみ)
- メニュー: マスタ → **変更ログ**
- フィルタ: テーブル / 操作(INSERT/UPDATE/DELETE) / ユーザー / 日付範囲 / フリー検索
- 詳細画面で **変更前/変更後** を並列表示 (黄色ハイライト = 変更あり)

## 10. トラブルシューティング

| 症状 | 対処 |
|------|------|
| ログインできない | `users` テーブル確認、`database/reset_passwords.php` で admin123 にリセット |
| PDF が白紙 | ブラウザコンソール確認、ロゴパス確認、GD/fileinfo 拡張有効化 |
| 名刺 OCR が動かない | ネット接続確認 (CDN)、ブラウザコンソールで Tesseract エラー確認 |
| タイ語文字化け | DB エンコーディング UTF8 確認、php.ini の mbstring 確認 |
| 500 エラー | PHP エラーログ (`php -S` のコンソール) 確認 |
| SQLSTATE[25P02] | 前のトランザクションが失敗。サーバー再起動 |
| 顧客コード重複 | `uniq_customer_code_active` 部分ユニーク制約でブロック |

## Step 4: `.env` 設定 & DB リストア (EC2 上)

```bash
# EC2 上で
cat <<'EOF' | sudo tee /var/www/pegasus_erp/.env > /dev/null
DB_HOST=<RDS_ENDPOINT を貼り付け>
DB_PORT=5432
DB_NAME=pegasus_erp
DB_USER=pegasus_admin
DB_PASS=<CloudFormation で設定したパスワード>
APP_ENV=production
APP_URL=http://<EC2_PUBLIC_IP>
AWS_REGION=ap-southeast-1
S3_BACKUP_BUCKET=pegasus-backups-889629667447
S3_UPLOAD_BUCKET=pegasus-uploads-889629667447
EOF

sudo chmod 600 /var/www/pegasus_erp/.env
sudo chown www-data:www-data /var/www/pegasus_erp/.env

# DB 接続テスト
source /var/www/pegasus_erp/.env
PGPASSWORD="$DB_PASS" psql -h "$DB_HOST" -U "$DB_USER" -d "$DB_NAME" -c "SELECT version();"
# → PostgreSQL 15.7 on ... が表示されれば OK

# リストア実行
PGPASSWORD="$DB_PASS" pg_restore \
    -h "$DB_HOST" -U "$DB_USER" -d "$DB_NAME" \
    -c --if-exists -v ~/pegasus_erp_20260417_125550.dump 2>&1 | tail -20

# テーブル数確認
PGPASSWORD="$DB_PASS" psql -h "$DB_HOST" -U "$DB_USER" -d "$DB_NAME" \
    -c "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema='public';"
# → 80件前後の数字が表示されれば OK
```

### `config/database.php` が環境変数を読むことを確認

PEGASUS ERP の `config/database.php` は既に `getenv()` 対応ですが、PHP-FPM プロセスに `.env` を読み込ませる必要があります。シンプルに `/var/www/pegasus_erp/public/index.php` の最上部で `.env` をロード:

```bash
sudo nano /var/www/pegasus_erp/public/index.php
```

最初の `<?php` 直後に追加:
```php
// Load .env if present
if (file_exists(__DIR__ . '/../.env')) {
    foreach (file(__DIR__ . '/../.env', FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES) as $line) {
        if (strpos(trim($line), '#') === 0 || !strpos($line, '=')) continue;
        [$k, $v] = explode('=', $line, 2);
        $k = trim($k); $v = trim($v, " \t\n\r\0\x0B\"'");
        if ($k && !getenv($k)) { putenv("$k=$v"); $_ENV[$k] = $v; }
    }
}
```

→ 保存して終了。

## 🔧 トラブル対処

### CloudFormation が失敗する
- **IAM リソース承認忘れ**: 再実行時に最後のチェックボックスを確認
- **DB パスワード文字制限**: 引用符・スペースは不可
- **キーペア未存在**: Step 1 を先に完了
- **AMI ID 古い**: 最新の Ubuntu 22.04 AMI に差し替え
  ```bash
  aws ec2 describe-images --owners 099720109477 \
    --filters 'Name=name,Values=ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*' \
    --query 'sort_by(Images,&CreationDate)[-1].ImageId' --region ap-southeast-1
  ```

### SSH 接続できない
- **パーミッション**: `icacls` で絞っていることを確認
- **SG の SSH ポート**: あなたの IP が許可されているか
- **EIP 変わってないか**: スタックの出力タブで再確認

### ブラウザで `502 Bad Gateway`
```bash
sudo systemctl restart php8.2-fpm nginx
sudo tail -30 /var/log/nginx/error.log
sudo tail -30 /var/log/php_errors.log
```

### DB 接続エラー (`connection refused`)
- SG `sg-pegasus-db` の Inbound に `sg-pegasus-web` があるか
- RDS のステータスが「利用可能」か
- `.env` のパスワードにタイポがないか

## 🛑 スタックを削除する場合 (検証後の片付け)

CloudFormation → スタック選択 → 「削除」

⚠️ **注意**:
- RDS は `DeletionProtection: true` なので **先に AWS コンソールで手動解除**
- `DeletionPolicy: Snapshot` なので DB の最終スナップショットは残る
- S3 バケットは `DeletionPolicy: Retain` なので **手動削除が必要**
